# InfoOly Track 数据导入模板

> 本文件为 InfoOly Track 网站的数据导入模板。管理员可以通过管理后台的"导入"功能上传此文件，系统将自动解析并更新数据库。

---

## 一、赛制介绍（可选）

在 `## 赛制介绍` 标题下，可以列出各种赛制说明。

格式：
- **赛制名称**：赛制说明文本

示例：
- **OI赛制**：无提交反馈、无实时排名。比赛期间提交代码后不显示任何结果，仅赛后统一评测...
- **IOI赛制**：有实时反馈、有实时排名...
- **ACM-ICPC**：以三人团队为单位，在5小时内共用一台计算机解决若干道算法难题...

---

## 二、战绩（比赛记录）

在 `# 战绩` 标题下，按时间层级记录比赛成绩。

格式规则：
- 第一级：`- YYYY/MM` → 比赛时间
- 第二级：`- 线上/线下赛 比赛名称` → 比赛类型和名称
- 第三级：`- 学生姓名 奖项/得分` → 学生成绩
- 嵌套列表可附加详细信息（得分、排名、Rating变化等）

示例：
- 2024/10
  - 线下赛csp-j
    - 马天成 二等奖
    - 党皓天 一等奖
    - 陆臻 二等奖
- 2026/05
  - 线上赛atcoder ABC460
    - 马天成
      - 得分：1650/2725，排名：366
      - Rank总排名51225->23870，Rating积分186->731

## 三、难度分级（知识点体系）

在 `### 难度分级` 标题下，定义知识点体系。

格式规则：
- 第一级：`- 数字(等级别名)` → 难度等级
- 第二级：`- 知识点名称` → 具体知识点

等级映射：
- 3 → CSP-J
- 4 → GESP 7级
- 5 → CSP-S
- 6 → USACO金
- 7 → 省选

示例：
- 3(csp-j)
  - set数据结构运用
  - 快速幂
  - 并查集
  - 基础dp
- 5(csp-s)
  - 匈牙利算法
  - 朴素分块
  - 莫队模板
- 7(省选)
  - HK算法
  - 莫队+位运算

## 四、知识点认证

在 `### 知识点认证` 标题下，记录学生已掌握的知识点及认证时间。

格式规则：
- 第一级：`- 学生姓名`（可用"-"连接，如 张-赫-桐）
- 第二级：`- 知识点名称 YYYYMM` → 认证时间（6位数字年月）

示例：
- 马-天-成
  - 匈牙利算法 202603
  - HK算法 202603
  - 朴素分块 202603
  - 莫队模板 202604
  - 状压DP 202605
- 党-皓-天
  - 匈牙利算法 202603
  - 带权二分图匹配KM算法 202603
  - 按位贪心 202604

## 五、任务板

在 `### 任务板` 标题下，记录每个学生的学习任务及完成状态。

格式规则：
- 第一级：`- 学生姓名`（可用"-"连接）
- 第二级：`- [x] 任务标题` → 已完成
- 第二级：`- [ ] 任务标题` → 未完成（待办）

示例：
- 张-赫-桐
  - [x] 匈牙利算法 TR01
  - [x] set数据结构 319,290,291
  - [x] 朴素分块 TR17
  - [ ] 贪心练习 NOIP01
  - [x] 状压DP练习 1102
  - [ ] gesp五六七八级 GESP01~05
- 马-天-成
  - [x] 匈牙利算法 TR01,TR02
  - [x] HK算法自主精研代码
  - [ ] 分块进阶 TR08,TR09,TR10
  - [x] 状压DP练习 1102

## 导入注意事项

1. 支持上传 `.md` 格式的 Markdown 文件
2. 各模块使用标准 Markdown 标题层级（`#`、`##`、`###`）标识
3. 使用 `-` 作为列表项标记
4. 任务板使用 `[x]` 和 `[ ]` 表示完成状态
5. 学生姓名可用连字符连接（如 马-天-成）或直接使用（如 马天成）
6. 系统会自动解析并导入到对应的数据库表中
